# agentgateway v0.1 — release notes

**Status:** release-blocker tracker closed (see [`PENDING_TODO.md`](../../PENDING_TODO.md)
§12). v0.1 ships a NATS-native MCP + A2A agentgateway with Tier-1
SpiceDB authorization, JWT validation, audit, and operator-grade
substrate provisioning (Helm + NSC + SpiceDB + JetStream manifests).

## What ships

### Protocol surface

- **MCP** — JSON-RPC 2025-06-18 over NATS request/reply.
  - `tools/call`, `resources/read`, `prompts/get`, `tools/list` filtering.
  - Stdio bridge via `mcp-nats-stdio`.
  - Static MCP backend, dynamic backend (registry-watch), virtual MCP
    federation (scaffold; cross-backend ignored test).
- **A2A** — v0.2.x.
  - `message/send` unary RPC (30-second default deadline).
  - `message/stream` SSE with pull-consumer back-pressure.
  - `tasks/{get,list,cancel,resubscribe}`.
  - Push notifications (`pushNotificationConfig/*`) with NATS DLQ
    and gateway-side mirror.
  - Audit envelopes at every deny/forward/deadline decision site.
- **ACP** — TrogonStack-native, session lifecycle, prompt/cancel,
  filesystem and terminal proxy, permission requests, extension
  methods.

### Security

- JWT validation (issuer + JWKS), trusted downstream headers on
  successful validation.
- Tier-1 declarative authorization via SpiceDB
  (`trogon/mcp_tool`, `trogon/mcp_resource`, `trogon/a2a_agent`,
  `trogon/a2a_task`).
- Tier-1 in-process rate limiter (ADR 0012). KV-sync Tier-2 deferred
  to v0.2.
- Redaction ruleset loaded from YAML (`replace`, `regex_replace`).
- NATS `$SYS.REQ.USER.AUTH` auth-callout subscriber
  (`a2a-auth-callout`), full implementation; wiring into the bridge's
  stub client is the v0.2 production cutover.

### Operations

- Health probe listener (`/healthz`, `/readyz`) on configurable
  `*_PROBE_LISTEN_ADDR`.
- OTel traces, metrics, logs (no Prom direct scrape — pull via OTel
  collector; ADR 0034 §3).
- Splunk HEC + Elastic ECS audit reshapers.

### Deploy

- `charts/agentgateway` Helm chart with bin-pluggable image and
  per-binary `values-<binary>.yaml` overlays.
- Opt-in `NetworkPolicy`, `PodDisruptionBudget`, `helm test`, and
  NATS pre-install provisioner Job.
- `charts/agentgateway-crds` stub for v0.2 Gateway-API CRDs.
- `devops/nats/` — every stream and KV bucket the runtime expects,
  plus `nats-server.example.conf` with `auth_callout` and JetStream
  blocks.
- `devops/nsc/` — operator + Account env templates, ACL templates
  for caller/gateway/registrar Users, and a self-contained bootstrap
  wrapper.
- `devops/spicedb/schema.zed` — reference Tier-1 schema.

### Supply chain

- `cargo-deny advisories|bans|licenses|sources` green.
  Eight RUSTSEC IDs ignored with documented rationale and SemVer
  Upstream-fix tracker (`deny.toml`).
- Multi-arch container builds (`linux/amd64` + `linux/arm64`) with
  SLSA provenance and SBOM.
- `release-please` covers every shipping crate.

## Explicit non-goals (Option A under ADR 0017)

The following upstream agentgateway feature areas are intentionally
**not** built for v0.1. They are tracked verbatim in `PENDING_TODO.md`
§1.4 and §1.5 so a future pivot has a starting list.

- **LLM gateway** — no unified OpenAI-compatible API, provider
  adapters, model aliasing, virtual keys, LLM rate limiting, prompt
  enrichment, guardrails, cost tracking.
- **Inference routing** — no Inference Gateway extensions,
  `InferencePool` API, KServe integration.
- **OIDC redirect ingress** — we validate JWTs, we don't act as an
  OAuth Relying Party.
- **API-key / Basic-auth ingress** — JWT/SVID only.
- **HBONE / Istio mesh transport** — not in the deployment substrate.

## Known limitations and v0.2 deferrals

Tracked in [`docs/roadmap/agentgateway-v0.2.md`](roadmap/agentgateway-v0.2.md):

- HTTPS ingress (`a2a-bridge` ships as `0.0.0-skeleton`).
- Multi-region routing wiring (`RegionRouter` type exists, not called).
- KV-sync rate limiter (ADR 0012 Tier 2).
- Real auth-callout production wiring (replaces `StubAuthCalloutClient`).
- WASM Tier-3 redaction CEL → WASM compile path.
- Admin write API + Gateway-API CRDs + xDS interop (v2 default per
  ADR 0017).
- Grafana dashboard manifests.
- Reference-deployments example tree (mirrors upstream `examples/`).

## Upgrade / install

- New install: `helm install agentgateway charts/agentgateway`.
- Substrate: apply manifests in `devops/nats/{streams,kv}/`,
  `devops/spicedb/schema.zed`, and (per tenant) the NSC bootstrap in
  `devops/nsc/bootstrap.sh`.
- See `charts/agentgateway/README.md` for multi-binary topology
  and `existingSecret` schema.
