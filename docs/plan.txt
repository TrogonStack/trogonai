  Crate nuevo: crates/trogon-cli/

    Cargo.toml                                                                  
    src/
      main.rs          ← clap + arranque

  Dependencias nuevas: rustyline = "14", acp-nats (ya en workspace), clap (ya en

  Comportamiento del REPL:
  - Conecta a NATS (default nats://localhost:4222, configurable via
  TROGON_NATS_URL)                                                              
  - Crea sesión ACP con cwd = std::env::current_dir()
  - Loop: leer línea → procesar @mentions y slash commands → enviar prompt →    
  stream de eventos → imprimir TextDelta en tiempo real                         
  - Ctrl+C envía cancel a la sesión activa                                      
  - Ctrl+D sale limpiamente cerrando la sesión                                  
  - Historial persistido en ~/.local/share/trogon/history

  19. Bridge stdio↔HTTP para servidores MCP
                                           
  Archivo: crates/trogon-cli/src/stdio_mcp_bridge.rs
                                                                                
  Problema: StoredMcpServer en trogon-acp-runner explícitamente no soporta      
  stdio. El bridge resuelve esto en el lado del CLI, sin tocar el servidor:     
                                                                                
  - El CLI lanza el proceso MCP stdio como hijo (ej. npx                        
  @modelcontextprotocol/server-filesystem ./)
  - Levanta un servidor axum en un puerto local aleatorio que hace proxy        
  JSON-RPC ↔ stdin/stdout del proceso hijo                                      
  - Registra el servidor en la sesión como StoredMcpServer::Http { url: 
  "http://127.0.0.1:<port>" }                                                   
  - Al terminar el CLI, mata el proceso hijo y libera el puerto
  - El runner en NATS no sabe nada de stdio; solo ve un servidor MCP HTTP normal
